\begin{abstract}
    We investigate whether Kolmogorov-Arnold Networks (KANs) can be integrated into graph neural networks (GNNs) to produce interpretable models of $n$-body physical systems. We introduce Graph KANs (GKANs), a hybrid architecture that replaces MLP components in message-passing GNNs with learnable spline-based edge functions. We evaluate GKANs on simulated $n$-body dynamics under several force laws and assess both predictive performance and interpretability through sparsity and network visualization. Our results are largely negative: GKANs are consistently harder to optimize than MLP baselines, and standard regularization techniques fail to produce interpretable sparse networks. We attribute this primarily to the interaction between spline-based edge functions and message passing, which amplifies gradient instability. Despite this, we find that physics-informed feature engineering can recover baseline GNN performance with a 99.5\% reduction in model size, and that smoother force laws are substantially easier to learn---suggesting that GKANs may still hold promise under the right inductive biases.
\end{abstract}

\section{Introduction}

Many scientific systems can be modeled as interacting objects governed by underlying physical laws. A key goal in machine learning for science is not only to predict these systems accurately, but also to recover interpretable structure from the learned model.

We study the $n$-body problem as a testbed for this goal. Given the initial states of several bodies, the task is to predict their future trajectories under pairwise forces. Prior work has shown that graph neural networks (GNNs) are a strong fit for this setting because they naturally represent objects as nodes and interactions as edges. In particular, \cite{cranmer2020discoveringsymbolicmodelsdeep} showed that a GNN trained on simulated $n$-body dynamics can be combined with symbolic regression to recover analytical force laws.

A limitation of this approach is that the GNN itself is not directly interpretable. The learned representation is still a black box, and symbolic regression is a separate post-hoc step. Recent work on Kolmogorov-Arnold Networks (KANs) suggests a possible alternative. KANs replace standard MLP-style node activations with learnable univariate functions on edges, which can make the learned model easier to inspect. \cite{liu2024kankolmogorovarnoldnetworks} showed that KANs can be effective while remaining more transparent than standard MLPs.

In this project, we ask whether this idea can be extended to relational physical systems. We introduce a hybrid Graph KAN (GKAN) architecture, where KAN layers replace MLP components inside a message-passing GNN. We evaluate this model on simulated $n$-body dynamics and compare it against standard GNN baselines, with a focus on both prediction quality and interpretability.  However, whether KANs integrate well with message passing is not obvious---the two components impose different structural demands on optimization, and our results reveal this tension in practice.

\section{Related Work}

This project builds on three main lines of work: graph-based physics learning, symbolic discovery, and interpretable neural architectures.

\cite{cranmer2020discoveringsymbolicmodelsdeep} is the closest prior work. They trained GNNs on simulated $n$-body systems and used symbolic regression to recover physical laws from learned interaction functions. Their results show that graph models can discover useful structure, but interpretability comes only after a separate extraction step.

Other work has strengthened graph models for physical simulation. \cite{thangamuthu2023unravellingperformancephysicsinformedgraph} compared several physics-informed GNNs across multiple dynamical systems, while Dynami-CAL GraphNet \parencite{sharma2025dynamicalgraphnetphysicsinformedgraph} adds conservation and symmetry constraints directly into message passing. These methods improve performance, but do not directly address whether the learned interactions are easy to interpret.

KANs provide a different route to interpretability. \cite{liu2024kankolmogorovarnoldnetworks} showed that KANs can represent scientific relationships through learnable univariate functions, and KAN 2.0 adds multiplicative structure that is especially relevant for physical equations. Related continuous-time work such as KAN-ODEs \parencite{Koenig_2024} and Graph KAN-ODEs \parencite{cappi2025unveilingactualperformanceneuralbased} further motivate combining KANs with dynamical systems and graph structure.

\section{Methodology}

We evaluate GKANs on synthetic $n$-body simulation data generated with a leapfrog integrator. The main dataset uses Newtonian gravity,

\[
a_{i} = G \sum_{j \neq i} m_{j} \frac{s_{j} - s_{i}}{\|s_{j} - s_{i}\|^{3}}
\]

with additional force laws for generalization tests. Each example consists of initial positions, velocities, and masses, with targets given by future trajectories. We compare a GNN baseline following \cite{cranmer2020discoveringsymbolicmodelsdeep} against a GKAN, trained on identical data with the same loss and optimizer. We evaluate both predictive performance on held-out simulations and interpretability through network sparsity and visualization of learned edge functions.

Our experiments address four questions: whether GKANs match GNN performance at comparable parameter counts; whether sparse regularization improves interpretability; whether physically motivated minimal architectures are sufficient; and whether these trends generalize across force laws.

\subsection{Baseline GNN}

We follow the baseline GNN architecture from \cite{cranmer2020discoveringsymbolicmodelsdeep}, a node MLP of $(110 \to 300 \to 300 \to 2)$ and a message MLP of $(10 \to 300 \to 300 \to 300 \to 100)$. Note that we compute the loss as the \textit{mean} over individual samples rather than the \textit{sum} (as in \cite{cranmer2020discoveringsymbolicmodelsdeep}). The model is trained without regularization using an L2 loss, optimized with Adam at a learning rate of $10^{-3}$.

\subsection{GKAN Architecture}

Our GKAN replaces the MLP components of the baseline GCN with KAN layers. Specifically, the message MLP is replaced with a KAN message network and the node update MLP with a KAN node network. Each KAN edge learns a univariate B-spline function with a fixed grid size of \texttt{G} and spline order \texttt{k}, giving it significantly more interpretable structure than a standard linear layer followed by an activation.

We expect the node KAN to learn how to aggregate messages into a final acceleration vector, while the edge KANs learn the pairwise interaction functions. We hypothesize that regularization will encourage sparsity and interpretability, and that smaller KAN architectures may still perform well if they capture the right inductive biases.

\subsection{Regularization}
Sparsification can be achieved in KAN layers through L1 regularization of weights, as in traditional MLP layers, though an additional entropy regularizer is often needed to make sufficiently sparse KANs \parencite{liu2024kankolmogorovarnoldnetworks}. This suite of experiments serves to both validate regularization patterns found in traditional KANs, as well as find ideal regularization hyperparameters to encourage sparse and interpretable networks. To do so, we run a sweep through different magnitudes of L1 regularization, entropy regularization, and both combined, comparing the models to each other, as well as a control with no regularization. Doing so helps identify the efficacy of different regularizations in creating sparser models. Additionally, a model utilizing sparse initialization is also trained, which forces the networks to be sparse before training.

Each GKAN network uses a node network of 3 layers $(7 \to 8 \to 2)$, a message network of 4 layers $(10 \to 12 \to 12 \to 2)$, and is trained over 50 epochs. Once all models are trained, they are compared in both training loss and validation loss against a test set, as well as visually compared for how sparse the networks are. The ideal regularization approach is determined by which hyperparameters provide a relatively sparse and interpretable network without sacrificing performance on the validation set.

\subsection{How small KAN we get?}

The Kolmogorov-Arnold (KA) representation theorem states that any continuous multivariate function can be decomposed into a finite number of univariate functions and addition operations \parencite{liu2024kankolmogorovarnoldnetworks}.

\[
f(x_{1 \ldots n}) = \sum_{i = 0}^{2n} \Phi_{i} \left( \sum_{j = 1}^{n} \phi_{i, j}(x_{j}) \right)
\]

This suggests the existence of models with significantly fewer parameters that can still represent the function we want to learn. In this section, we design and evaluate several smaller models to determine if the GKAN can perform well without extreme overparameterization.

To design a smaller model, we work backwards from a proposed ``feasible route'' of function compositions and additions that yields the desired force law. In other words, we set layer widths and depths based on the number of arithmetic steps needed to decompose the desired force law into univariate functions by hand. We then observe whether the model is capable of recovering the desired force law by ``undoing'' our decomposition.

To illustrate this process, Figure \ref{fig:feasible-route-small} is one example of a feasible route. In this example, the message network consists of 4 layers ($10 \to 6 \to 3 \to 4 \to 2$) because the acceleration vector can be computed in 4 arithmetic steps; the node update network consists of just 1 layer ($7 \to 2$) because it only needs to aggregate acceleration vectors from the messages.

By providing these as inputs, we can eliminate 1 layer.

\subsection{Different Force Functions}

Utilizing the derived small model above, we also test the GKAN's ability to learn different force functions. In particular, we replace the gravitational force with the following three force functions:

\[
a_{\text{lin}} = G \sum_{j \neq i} m_{j} \frac{s_{j} - s_{i}}{\|s_{j} - s_{i}\|^{2}}, \quad
a_{\text{hooke}} = G \sum_{j \neq i} \frac{s_{j} - s_{i}}{m_i}, \quad
a_{\text{nice}} = G \sum_{j \neq i} \exp(\|s_j - s_i\|^2) \frac{s_j - s_i}{m_i}
\]

$a_{\text{lin}}$ is a linear gravity function that has a $1/r$ fall-off instead of $1/r^2$. $a_{\text{hooke}}$ is a Hooke's law function that has a linear relationship with distance. $a_{\text{nice}}$ is an arbitrary function with a nice gradient.

We also ran similar regularization experiments on the different force functions to see if the same patterns hold.

\section{Results and Discussion}

\subsection{Baseline GNN}
The baseline GNN model achieves a training loss of $1.783 \times 10^{-2}$ and a validation loss of $1.289 \times 10^{-2}$ after 50 epochs. This serves as a reference point for evaluating the performance of the GKAN models.

\subsection{Regularization}
Unregularized training loss and validation loss for each combination of L1-norm and entropy regularizations in Table \ref{tab:regularization} display marginal differences between values. The largest gap in validation losses between two of the models is $0.08\times 10^{-1}$, comparing the control ($0$ L1-norm, $0$ entropy) to the maximal regularization ($1\times 10^{2}$ L1-norm, $1\times 10^{2}$ entropy). The sparse initialization is the only model to not follow this trend, with training loss $3.95\times 10^{-1}$ and validation loss $3.57\times 10^{-1}$.

None of the networks produced by each model are interpretable through visual inspection, even with $1.0\times 10^{2-2}$ regularization, which did not have a significantly sparser network compared to the control. Node and message networks are available in the appendix (Figure \ref{fig:reg_nets}).

From \cite{liu2024kankolmogorovarnoldnetworks}, we expected that indivdually L1-norm and entropy would be insufficient. However, contrary to our expectations, even the combined regularization did not produce significantly sparser networks or better validation losses. The differences between losses in the experiments is easily explained as noise, and none of the regularization techniques encouraged sparse networks. This may be explained by:

(1) Sparse Learning Difficulty. Learning the sparse representation of the gravitational $n$-body is difficult. Using L1 loss on the difference in output accelerations may not be sufficient the model to learn ``physically'' important structure (eg. energy conservation, symmetries, etc.) that would encourage sparsity.  We may find better performance by incorporating physical structure directly into the loss function. Physics-informed GNNs such as Dynami-CAL GraphNet \parencite{sharma2025dynamicalgraphnetphysicsinformedgraph} enforce conservation of linear and angular momentum through equivariant message passing, effectively constraining the hypothesis space to physically consistent solutions. In our setting, such constraints could encourage sparser, more interpretable representations while also improving optimization stability and long-term rollout accuracy.

(2)Constant regularization. Constant regularization does not allow for balance between learning and sparsity. The experiments did not increase regularization magnitudes over the learning period, which may have done a better job of letting the model optimize for accurate predictions first, then learn a sparse representation after.

The \texttt{small} GKAN model without any computed features performs about 1 order of magnitude worse compared to the baseline GNN. This alone is quite impressive when considering the difference in model size.

Furthermore, by introducing simple computed features such as $\Delta x$ and $\Delta y$ (which would not be unreasonable to include, even without \textit{a priori} knowledge of the desired force law), we observe significantly improved performance on par with the baseline---with the additional benefit of a 99.5\% reduction in model size. This suggests that the GKAN could be a highly efficient architecture for domain-informed use cases.

Looking at the visualized networks in Figures \ref{fig:message-network-small}--\ref{fig:node-update-network-small}, it seems that the \texttt{small} model with no feature engineering did not learn as effectively. The message and node update networks are dense, which suggests that the model did not learn to ignore features such as velocity which are not relevant to the desired force law. The introduction of computed features (obtensibly) acts as a  sufficiently strong inductive bias---the models with feature engineering have sparser networks which emphasize the relevant mass- and displacement-related features. In conjunction with the improvement in the training and validation loss, this supports the idea that computed features can increase the expressive power of a model despite adding no new information \parencite{liu2024kan20kolmogorovarnoldnetworks}.

The utility of the GKAN in terms of interpretability is still unclear. A cross-section visualization (perturbing one input at a time) of the \texttt{small\_rel\_pos} model as shown in Figure \ref{fig:cross-section-small-rel-pos} indicates that the model learned approximately correct mathematical relationships overall. However, this type of analysis is equally possible with traditional GNN architectures.

The unique property of the GKAN is that it is possible to perform symbolic regression per-layer, then compose the symbolic expressions analytically. Unfortunately, in our case, while the overall result was reasonable, the smaller models did not learn the exact per-layer transformations we had hoped for (as outlined in our feasible routes). This is not surprising---there may be an infinite number of valid decompositions for any given function---but it complicates interpretability, as the size of the symbolic expression increases exponentially with network depth. Ultimately, we were not able to produce interpretable symbolic expressions for the learned force laws.

The relative training and validation loss decreases the GKAN models on 10 epochs of training are in Table \ref{tab:results-different-force}. In terms of reguarlization results, we see the same patterns with little difference in sparsity. With the only exception being Hooke's law, which has a significantly more sparse network (see figure \ref{fig:different_force_msg_models} in the appendix).

We observe a clear trend: GKANs train more effectively on smoother, better-conditioned force laws. Across all four functions, both training and validation loss reductions increase as the target interaction becomes less singular and more regular.

Gravity performs worst, with training loss decreasing by less than $7\%$ over 10 epochs. This is unsurprising: the $1/r^2$ dependence introduces near-singular behavior at small separations, which is difficult for B-spline edges to represent efficiently, as their piecewise polynomial basis approximates smooth functions well but struggles with sharp or unbounded derivatives~\parencite{DEBOOR197250}. Replacing this with the $1/r$ linear gravity law already yields a substantial improvement, with training loss dropping by $\sim 31\%$, suggesting that reducing the degree of singularity meaningfully aids optimization.

Hooke's law improves further still, achieving a $\sim 60\%$ training loss reduction. Because the interaction is linear in displacement with bounded derivatives, the model requires less functional composition and fewer active spline regions to represent it accurately. This is also reflected in the regularization results, where Hooke's law (figure~\ref{sub@fig:diff_hooke_msg}) is the only force function that produces a noticeably sparser network. The ``nice'' force function, despite being nonlinear, achieves comparable relative loss reductions ($\sim 45\%$ on validation) and maintains the lowest absolute losses throughout training---likely because its smooth exponential profile avoids the distance-magnitude and reciprocal-scaling terms that make gravity challenging.

In combination, these results suggest that GKAN optimization is sensitive to the analytic structure of the target force law. Interpretability and sparsity also benefit from smoother targets: simpler force laws not only converge faster, but compress more naturally into compact GKAN representations.

\subsection{KAN't Optimize}

Beyond interpretability, a persistent challenge in our experiments was optimization. GKANs were consistently harder to train than MLP-based GNN baselines: loss decreased slowly, converged to worse minima, and was sensitive to initialization. This was surprising given that \cite{liu2024kankolmogorovarnoldnetworks} and \cite{liu2024kan20kolmogorovarnoldnetworks} demonstrate effective KAN training on gravitational systems---however, their setting trains a single KAN directly on force inputs and outputs, without the added complexity of message passing.

We suspect message passing is the key differentiator. Errors in learned edge functions propagate across the graph, amplifying the gradient instability that spline-based functions already exhibit. This is consistent with known pathologies in deep GNNs such as oversmoothing and oversquashing \parencite{arroyo2025vanishinggradientsoversmoothingoversquashing}. We tested Adam, L-BFGS, warmup schedules, and alternating optimization, but none consistently improved training, suggesting that this difficulty may be intrinsic to the GKAN architecture in message-passing settings.


\section{Conclusion}

We introduced GKANs, an architecture combining KAN layers with GNNs, and evaluated them on $n$-body dynamics. Our results were largely negative: GKANs were consistently harder to optimize than MLP-based GNN baselines, regularization failed to produce interpretable sparse networks, and training loss had mediocre decreases. This was surprising given that prior work demonstrates KANs can recover physical laws effectively---though notably in simpler, non-relational settings without message passing. Our experiments do leave us hopeful: GKANs trained with feature engineering showed good performance with a $99.5\%$ reduction in model size, and smoother force laws were substantially easier to learn, hinting that the architecture has real potential under the right conditions.

Many directions remain unexplored. Pruning low-weight edges during training, physically-informed loss functions enforcing conservation laws, and decoupled optimization schedules that prioritize accuracy before sparsity are all natural next steps. Resolving the gradient degradation introduced by message passing---perhaps through equivariant or symmetry-constrained architectures---may be the most critical open problem for making GKANs practical.

So KAN it learn? \textit{Kind of}.

\subsection{Code and Experiments}

All of our code and experiments are available at \url{https://github.com/Hanson-Sun/3body-gkan} (not Jupyter notebooks thankfully). The repository includes the training and evaluation code for all models, as well as all the information to reproduce our experiments. Specific experiment configurations can be found in the \texttt{experiments.yaml} file.

There are several additional features that we implemented but did not have time to test fully and include in our report. These include:
\begin{itemize}
    \item Importance score based pruning of final models to help visualization
    \item MultKan support, which is a variation of KAN that includes multiplication nodes in addition to addition and function composition.
    \item Reguarlization schedules that increase regularization magnitudes over the course of training to encourage learning first, then sparsity.
    \item Incomplete support for Hamiltonian GNNs.
    \item Large scale GKAN experiments with ~500k parameters. 
\end{itemize}

The repository also includes animated visualizations showcasing the rollouts (forward simulation) for selected models, which are linked in Table \ref{tab:rollout-links}.
